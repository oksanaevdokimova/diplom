# Спецификация JSON-сообщений
Формат обмена между приложением на ПК и контроллером робота. Одно сообщение — один объект JSON.

## Типы сообщений
Поле `type` (строка) задаёт вид сообщения:
`cmd` - команда от ПК к контроллеру
`ack` - подтверждение или ответ контроллера на команду
`telemetry` - телеметрия: текущее состояние с контроллера

## Обязательные поля
`axis` (целое число, ≥ 0) - номер оси (0 — первая ось, 1 — вторая и т.д.).
`position` (число) - для `cmd` — заданное положение, для `ack` и `telemetry` — фактическое или подтверждённое положение
`status` (строка) - состояние или результат
`error_code` (целое число) - `0` — без ошибки, иначе код ошибки

## Поле `status` по типам
`type`: `cmd` — команда, например перемещение (`status` — `"move"` (перейти в `position` по оси `axis`))
`type`: `ack` — ответ на команду (`"ok"`, `"rejected"`, `"busy"`)
`type`: `telemetry` — периодический снимок состояния (`"running"`, `"idle"`, `"fault"`)

## Примеры

Команда (`cmd`)
```json
{
  "type": "cmd",
  "axis": 0,
  "position": 150.0,
  "status": "move",
  "error_code": 0
}
```

Подтверждение (`ack`)
```json
{
  "type": "ack",
  "axis": 0,
  "position": 150.0,
  "status": "ok",
  "error_code": 0
}
```

Телеметрия (`telemetry`)
```json
{
  "type": "telemetry",
  "axis": 0,
  "position": 148.5,
  "status": "running",
  "error_code": 0
}
```
