# Прошивка контроллера

Скетч `robot_controller/robot_controller.ino` — ESP32 + **2 шаговых двигателя** через драйверы **STEP/DIR** (A4988, DRV8825, TMC2209 в режиме STEP/DIR). Протокол совпадает с приложением и симулятором.

## Механизмы

| ID | Имя в приложении | Пины ESP32 (по умолчанию) |
|----|------------------|---------------------------|
| 0 | Поворот | STEP 25, DIR 26, ENABLE 27 |
| 1 | Захват | STEP 32, DIR 33, ENABLE 14 |

Пины можно изменить в начале файла `.ino`.

## Схема (один драйвер)

```text
ESP32          Драйвер A4988          Мотор
GPIO STEP  ──► STEP
GPIO DIR   ──► DIR
GPIO EN    ──► ENABLE (LOW = включён)
               VMOT ── питание 12 V (по даташиту)
               GND  ── общая земля с ESP32
               1A, 1B ── обмотки шагового двигателя
```

Микрошаги и ток — подстроите перемычки/потенциометр на модуле драйвера.

## Команды из приложения

| Поле | Значение для шаговика |
|------|------------------------|
| `position` | целевая позиция в **шагах** (абсолютная, от нуля при включении) |
| `speed` | условная скорость → пересчёт в шаги/с (≈ speed × 80) |
| `move` | разгон до цели, по приходу — телеметрия `stopped` |
| `stop` | мгновенная остановка, удержание позиции |

## Библиотеки (Arduino IDE)

1. **ArduinoJson** 7.x  
2. **AccelStepper** (Mike McCauley)

Плата: **ESP32** (Dev Module), скорость UART **115200**.

## Подключение к приложению

```json
"active_transport": "usb",
"usb": { "port": "/dev/cu.usbserial-…", "baudrate": 115200 }
```

На Windows: `"port": "COM7"`. Закройте Serial Monitor в IDE перед запуском приложения.

## Безопасность

- Питание моторов **отдельно** от USB ESP32 (общая только **GND**).
- Поставьте концевики/ограничение хода в механике; в прошивке лимиты по шагам не заданы.
