# Система управления исполнительными механизмами робота

Desktop-приложение оператора (PySide6): подключение к контроллеру, список механизмов, команды move/stop, телеметрия и диагностика.

## Требования

- Python 3.9+
- Зависимости: `pip install -r requirements.txt`
- Рекомендуется виртуальное окружение: `python3 -m venv .venv && .venv/bin/pip install -r requirements.txt`

## Запуск

```bash
.venv/bin/python -m app.main
```

## Подключение к контроллеру

1. Укажите параметры канала в панели подключения (COM-порт, IP:порт или MQTT) и сохраните конфигурацию.
2. Нажмите **Подключить** → после ping/pong статус «Подключено», запрос списка механизмов.
3. Выберите механизм — телеметрия и диагностика фильтруются по ID.
4. Убедитесь, что прошивка контроллера использует тот же JSON-протокол (одна строка JSON на сообщение, см. `firmware/`).

## Конфигурация

Файл `config/config.json` (создаётся при сохранении из панели подключения) накладывается на `config/default_config.json`.

Ключевые поля:

| Поле | Описание |
|------|----------|
| `timeout_seconds` | Таймаут ping/pong и ожидания response |
| `default_speed` | Скорость по умолчанию, если поле пустое |
| `active_transport` | `usb` / `wifi` / `gsm` |

## Структура проекта

| Каталог | Назначение |
|---------|------------|
| `app/` | Точка входа |
| `config/` | Загрузка и валидация конфигурации |
| `protocol/` | Сообщения, типы, механизмы |
| `transport/` | USB, TCP, MQTT |
| `core/` | Связь, команды, разбор сообщений |
| `ui/` | Главное окно и панели |
